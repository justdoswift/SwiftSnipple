# Plan 08-03 — Full shadcn-svelte Convergence

## Goal

把 `/studio` 之外仍然残留的公开站自定义交互壳层继续收口到 `shadcn-svelte + Tailwind`，让整站真正共享同一套按钮、卡片、标签、Tabs 与输入体系，而不是继续靠页面级私有样式维持另一套视觉。

## Scope

- 公开站主路由：
  - `apps/web/src/routes/+layout.svelte`
  - `apps/web/src/routes/+page.svelte`
  - `apps/web/src/routes/explore/+page.svelte`
  - `apps/web/src/routes/snippets/[id]/+page.svelte`
- 共享业务组件：
  - `CopyActionButton`
  - `FacetChips`
  - `CodeBlock`
  - `PromptBlock`
  - `SnippetCard`
  - `SnippetPreviewMedia`
- 全局样式层：
  - 删除已不再需要的 `liquid-glass` / `glass-panel` / `content-surface` / `studio-surface` 工具样式

## Deliverables

1. 公开站控件层统一回到 `shadcn-svelte` 共享组件
2. 公开站页面级私有 `<style>` 基本移除，仅保留 `ui/` 组件自身样式
3. `Studio` 与公开站一起通过 `svelte-check` 与 Playwright 回归

## Verification

- `pnpm -C apps/web run check`
- `pnpm -C apps/web run test:e2e -- discovery.spec.ts`
- `pnpm -C apps/web run test:e2e -- studio.spec.ts`
- `rg` 复查公开站与共享组件，确认排除 `ui/` 后不再保留私有 `<style>` 与旧 surface 工具类
